Что такое Стринги Алисы?

Эта сборка — результат бескомпромиссной войны с телеметрией, фоновыми службами и скрытыми слежками Яндекс Браузера. Здесь нет ничего лишнего. Только браузер. Без докладов «наверх».

Смерть телеметрии

Полное отключение всех видов сбора данных. Яндекс не узнает, какие сайты вы посещаете. Metrics ID обнуляется при каждом запуске.

Невидимость в системе

После закрытия браузера не остаётся никаких следов — ни фоновых процессов, ни записей в реестре, ни служб в системе.

Максимальная скорость

Удалены все обновления, проверки лицензий и фоновые задачи. Никаких service_update.exe, browser_wer.dll и прочего мусора.

5 режимов запуска

От обычного браузера с синхронизацией до полной анонимности с блокировкой 400+ доменов в hosts и режимом инкогнито.

Портативность

Носить с собой на флешке. Работает на любом компьютере без установки. Все данные профиля — в папке data/, а не в AppData.

Арсенал очистки

Удаление 8 служб Яндекса, 12 задач планировщика, блокировка 400+ доменов, политики реестра, экспорт настроек в .reg.

Установка

Шаг 1: Установите оригинальный Яндекс Браузер

Скачайте и установите Яндекс Браузер с официального сайта: browser.yandex.ru

Почему это нужно? Лаунчер использует оригинальные файлы браузера из %USERPROFILE%\AppData\Local\Yandex\YandexBrowser\Application для создания портативной копии.

Шаг 2: Скачайте сборку

Скачайте архив с раздела загрузки или с GitHub Releases. Внутри — лаунчер и portapp.json.

Шаг 3: Распакуйте архив

Распакуйте архив в любую папку (например, C:\Apps\Alices-Thong\ или на флешку).

Шаг 4: Запустите лаунчер и нажмите «Портативная установка»

Запустите Launcher.exe. Перейдите на вкладку «Установка» и нажмите «Портативная установка». Лаунчер скопирует файлы браузера в папку app/, переименует browser.exe → chrome.exe

Шаг 5: Готово!

Путь на вкладке «Запуск» обновится автоматически. Теперь можно удалить оригинальный Яндекс Браузер — портативная копия работает независимо.

1. Удаление служб Яндекса (sc stop + sc delete)

8 служб, которые Яндекс устанавливает в систему. Все они останавливаются и удаляются при запуске.

YaTransmgr # Транспортный менеджер
YandexProtectService # "Защита" — читай слежка
YandexDiskSync # Синхронизация с Яндекс Диском
YandexBrowserUpdater # Обновление браузера
YandexBrowserCrashHandler # Отправка крашей
YandexBrowserExtensionUpdater # Обновление расширений

2. Зачистка планировщика (schtasks /delete /f)

12 задач, которые Яндекс создаёт в планировщике Windows для фоновой активности.

YandexUpdaterTask YandexBrowserUpdateTask

3. Политики реестра

Применяются в HKLM\SOFTWARE\Policies\Yandex\YandexBrowser. Альтернативный путь с пробелом — тоже.

BackgroundModeEnabled = 0 # Никакого фона
ScreenshotsEnabled = 0 # Никаких скриншотов
RestoreOnStartup = 4 # Всегда чистая вкладка
UpdateDefault = 0 # Никаких обновлений
MetricsReportingEnabled= 0 # Никаких метрик
SessionRestoreAutoRestore=0 # Никакого восстановления
SyncDisabled = 1 # (режимы 2-4) Нет синхронизации

5. Флаги запуска браузера

Браузер запускается с набором флагов, которые отключают телеметрию на уровне Chromium.

Базовые флаги (все режимы, кроме 5):

--no-default-browser-check # Не проверять браузер по умолчанию
--no-first-run # Пропустить экран первого запуска
--no-pings # Не пинговать Яндекс
--no-report-upload # Не отправлять отчёты
--no-crash-upload # Не отправлять краши
--disable-background-mode # Запрет фоновой работы
--disable-background-networking # Нет фоновых сетевых запросов
--disable-background-timer-throttling # Нет троттлинга
--disable-breakpad # Нет краш-репортов
--disable-crash-reporter # Отключить краш-репортёр
--disable-dev-shm-usage # Отключить /dev/shm
--disable-logging # Нет логов
--disable-metrics # Нет метрик
--disable-metrics-reporting # Нет отчётов метрик
--disable-session-crashed-bubble # Нет пузыря о краше
--disable-usage-statistics # Нет статистики использования
--no-sandbox # Отключить sandbox (портативность)
--restore-on-startup=0 # Не восстанавливать вкладки
--silent-debugger-extension-api # Тихий дебаггер
--test-type # Тестовый режим
--ignore-certificate-errors # Игнорировать ошибки сертификатов
--force-dark-mode # Тёмная тема принудительно

Режимы 3 и 4 — отключение Алисы:

--disable-features=Screenshots,AutomaticTabDiscarding,
BackgroundModeEnabled,SessionRestoreAutoRestore,
YandexAliceAssistant,YandexVoiceAssistant,
VoiceActivation,AliceAssistant

Режим 4 — инкогнито:

--incognito

6. Экспорт реестра + очистка

Перед каждым запуском (режимы 1-4) лаунчер делает бэкап реестра, а затем вычищает его:

# Экспорт (бэкап)
reg export "HKCU\Software\Yandex\YandexBrowser" AlicesThong.reg
reg export "HKU\*SID*\Software\Yandex\YandexBrowser" AlicesThong_HKU.reg

# Очистка
reg delete "HKCU\Software\Yandex" /f
reg delete "HKU\*SID*\Software\Yandex" /f

.reg-файлы сохраняются в папке с лаунчером — можно в любой момент вернуть настройки обратно.

7. Лаунчер (Python, customtkinter)

Написан на Python с графическим интерфейсом на customtkinter. Компилируется в Launcher.exe через PyInstaller. Вкладки:

🚀 Запуск — 5 режимов с логом операций
📦 Установка — создание портативной копии из оригинального Я.Браузера
🛡 Блокировка — выборочная блокировка hosts-доменов
🗑 Браузер — установка/удаление Яндекс Браузера из системы

8. PortApps — AlicesThong.exe

AlicesThong.exe (не путать с Launcher.exe) — это файл от PortApps. Он запускает app/chrome.exe как портативное приложение. PortApps создаёт временный профиль в data/, перенаправляет реестр и изолирует браузер от системы. Launcher.exe управляет этим процессом.
